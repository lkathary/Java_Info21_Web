Вывод TransferredPoints

Функция выводит таблицу в человекочитаемом виде - ник пира 1, ник пира 2, количество переданных пир поинтов. Количество отрицательное, если пир 2 получил от пира 1 больше поинтов.

Получение таблицы

Возвращает таблицу вида: ник пользователя, название проверенного задания, количество полученного XP. В таблицу включать только задания, успешно прошедшие проверку (определять по таблице Checks).

Пиры, которые провели в кампусе целый день

Функция определяюет пиров, которые не выходили из кампуса в течение всего дня. Параметры функции: день, например 12.05.2022. Возвращает только список пиров.

Успешные и неуспешные проверки

Найти процент успешных и неуспешных проверок за всё время. Формат вывода: процент успешных, процент неуспешных.

Изменения в TransferredPoints

Вывод изменений в количестве пир поинтов каждого пира по таблице TransferredPoints. Результат выводится отсортированным по изменению числа поинтов.
Формат вывода: ник пира, изменение в количество пир поинтов.

Изменения пир пойнтов

Вывод изменения в количестве пир поинтов каждого пира по таблице, возвращаемой функцией №1. Результат выводится отсортированным по изменению числа поинтов.
Формат вывода: ник пира, изменение в количество пир поинтов.

Самое проверяемое задание

Вывод самого часто проверяемого задания за каждый день. При одинаковом количестве проверок каких-то заданий в определенный день, вывести их все.

Длительность последней проверки

Определение длительности последней P2P проверки. Под длительностью подразумевается разница между временем, указанным в записи со статусом "начало", и временем, указанным в записи со статусом "успех" или "неуспех".

Выполнившие все задания

Таблица всех пиров, выполнивших весь заданный блок задач и дату завершения последнего задания. Параметры процедуры: название блока, например "CPP". Результат выводится отсортированным по дате завершения.

К кому идти на проверку?

Определяет, к какому пиру стоит идти на проверку каждому обучающемуся. Определение происходит  исходя из рекомендаций друзей пира, т.е. нужно найти пира, проверяться у которого рекомендует наибольшее число друзей.

Процент пиров

Показывает процент пиров, которые:
Приступили к блоку 1
Приступили к блоку 2
Приступили к обоим
Не приступили ни к одному.

Самые дружелюбные

Определяет N пиров с наибольшим числом друзей. Параметры процедуры: количество пиров N. Результат выводится отсортированным по колличеству друзей.

Проверка на ДР

Показывает процент пиров, которые когда-либо успешно проходили проверку в свой день рождения. Также определите процент пиров, которые хоть раз проваливали проверку в свой день рождения.

Суммарный опыт

Определяет количество XP, полученное в сумме каждым пиром. Если одна задача выполнена несколько раз, полученное за нее количество XP равно максимальному за эту задачу.

Пиры, сдавшие два задания

Определяет всех пиров, которые сдали заданные задания 1 и 2, но не сдали задание 3. Параметры процедуры: названия заданий 1, 2 и 3.

Предшествующие задачи

Определяет сколько задач нужно выполнить, исходя из условий входа, чтобы получить доступ к текущей.

Удачный день

Определяет "удачные" для проверок дни. День считается "удачным", если в нем есть хотя бы N идущих подряд успешных проверки. Параметры процедуры: количество идущих подряд успешных проверок N.

Самый умный

Определяет пира с наибольшим числом выполненных заданий.
Формат вывода: ник пира, число выполненных заданий.

Самый опытный

Определяет пира с наибольшим количеством XP.
Формат вывода: ник пира, количество XP.

Солнце еще высоко

Определяет пира, который провел сегодня в кампусе больше всего времени.

Точность - вежливость королей

Находит пиров, приходивших раньше заданного времени не менее N раз за всё время. Параметры процедуры: время, количество раз N.

Мне больно видеть белый свет

Находит пиров, выходивших за последние N дней из кампуса больше M раз. Параметры процедуры: количество дней N, количество раз M.

Король ночи

Определяет пира, который пришел сегодня в кампус последним.

Я только на минутку

Находит пиров, которые выходили вчера из кампуса больше чем на N минут. Параметры процедуры: количество минут N.

Ранние входы

Определяет для каждого месяца, сколько раз люди, родившиеся в этот месяц, приходили в кампус за всё время (общее число входов). Для каждого месяца рассчитывается, сколько раз люди, родившиеся в этот месяц, приходили в кампус раньше 12:00 за всё время (число ранних входов). Для каждого месяца подсчитывается процент ранних входов в кампус относительно общего числа входов.
